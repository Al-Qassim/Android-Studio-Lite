## Confirmed decisions

| # | Topic | Decision |
|---|---|---|
| 1 | DI | **Koin** |
| 2 | Project files on disk | **App-private directory** (sandbox under app storage) |
| 3 | Build provider (v0.1) | **Fake `BuildService`**: timed phase progress, then a **bundled premade APK** → install UI. Real **GitHub Actions** provider later behind the same API |
| 4 | Editor ↔ Files | **`editor:data` may depend on `files:api`** for load/save |
| 5 | Shared types | **Per-feature `:model` modules** (no `:core:model`) |
| 6 | Fake build UX | **Timed progress** (`Queued → Uploading → Building → Downloading → ReadyToInstall`) then premade APK |
| 7 | Premade APK location | **Bundled asset** in `:feature:buildapk:data` (or `:app`) resources/`assets/` |
| 8 | Project metadata | **Room / SQLite** |
| 9 | Room ownership | **Feature tables/DAOs in `:feature:*:data`**; **`:integration:database`** assembles `RoomDatabase` |
| 10 | Navigation | **`:integration:navigation` owns the IDE nav graph** (nested `NavHost`); `:app` hosts it + permissions / install |
| 11 | Create-project template | **Fuller Gradle skeleton** (closer to a real Android Studio empty Compose project), even though fake build ignores most of it |
| 12 | Premade sample APK | **Honest demo APK** — fixed sample package; UI/docs state it is not built from the user’s project sources yet |
| 13 | Project list API | **`ProjectService.observeProjects(): Flow<List<Project>>`** (architecture contract) |
| 14 | Koin layout | **Per-feature `:di` modules**; `:integration:di` includes them + `:integration:database`; **`:app` only starts Koin** |
| 15 | Fake build duration | **Short demo** — ~5–8s total across phases (e.g. 1s / 1.5s / 3s / 1.5s) |
| 16 | Template file set | **Standard empty-Compose set** — settings/root/app Gradle, wrapper *properties* only (no wrapper jars), manifest, MainActivity + theme, basic `res/`, `.gitignore` |
| 17 | Delete project | **Metadata + wipe project directory**; force-close editor session if that project is open |
| 18 | Unsaved editor on leave | **Confirm dialog** — Save / Discard / Cancel |
| 19 | IdeCoordinator | **Nav-graph only for v0.1** — no `IdeCoordinator` façade; add later if a second entry point appears |
| 20 | Test strategy | **Light** — unit tests for pure domain (sandbox rules, name validation, fake build phase machine); no UI/instrumented requirement for v0.1 |
| 21 | App ↔ design system | **`:designsystem` leads** — shared look/tokens live in code (`docs/design-system.md`). Figma track archived (`archive/figma/`); do not sync UI to Figma. |

## Module sketch (v0.1 extras beyond architecture.md)

```text
:feature:*:model / :api / :data / :presentation / :di
:feature:settings:api / :presentation / :di   # Settings UI; uses auth for login/session
:integration:database   # RoomDatabase assembly (feature entities/DAOs)
:integration:di         # aggregates feature + database Koin modules
:integration:navigation # IDE NavHost / product navigation
:app                    # start Koin, ApkInstaller, permissions
:designsystem           # exists
```

---

## Suggested implementation order

1. **Gradle skeleton** — include modules, Koin start in `:app`, feature `model/api/data/presentation/di` shells, `:integration:database`
2. **Projects** — Room metadata + app-private roots + empty-Compose template + `ProjectsScreens` + delete wipes FS
3. **Files** — sandbox FS ops + `FilesScreens`
4. **Editor** — session + `EditorScreens`; load/save via `files:api`
5. **Build (fake)** — timed `BuildService` + bundled honest demo APK + `BuildScreens` + `ApkInstaller`
6. **Integration** — IDE nav graph wiring Projects → Files → Editor → Build → install
